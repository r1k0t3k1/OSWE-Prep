Dashboard

You can associate different services from different host so you do not have to look your self for a specific correlation, such as if a host group has a ram usage over 90% you buy new hardware for this host group.

Basic Monitoring

Hosts

A host is a device connected to a network.

Services

A service is command with which you retrieve informations from a host.

Browser

Here you can browse your nodes.The browser will give you an overview over the current host and service status of all host and services in your current node.

Hosttemplates

Use host templates for easier host creation.

Servicetemplates

Use service templates for easier service creation.

Servicetemplategroups

Use service template groups to append not present services to a host or a host group in an easy way.

Hostgroups

Host groups contain a group of hosts. You can assign all hosts to an action by only choosing the group.

Servicegroups

service groups contain a group of services. You can assign all services to an action by only choosing the group.

Contacts

Contacts contain information whom to notify in case of a particular state of a host or service.

Contactgroups

Contact groups contain a group of contacts. You can assign all contacts to a notification by only choosing the group.

Calendar

You can use the calendar to configure your holidays where the monitoring software will not send any notifications.

Timeperiods

Timeperiods contain a timespan for a regular week in which the monitoring of a service or host takes place.

Commands

A command executes as a nagios terminal command from the user nagios on your server.

Tenants

Tenants contain information about user permissions to Nagios objects.

Nodes

Organize your tenant structure.

Locations

Locations contain information about a specified location of a tenant node.

Grafana

You can generate graphs with data from different services.

Downtimes

You can configure a downtime for hosts and services such as for a planned maintenance. In the time of the maintenance the hosts or services are not tracked by your monitoring system.

Logentries

Logentries contain logs, wich you can look through and filter or search wich logs you want to see.

Notifications

Notifications will show you host or service notifications in a timespan.

Performance Info

Here you get an overview over your system preformance.

Expert Monitoring

User Defined Macros

Intended for system paths or specific system command line options, to make your Nagios configuration more reusable in other system environments.

Host Escalations

Host escalations are optional follow-up notifications, that generates if a state change is not revoked in time.

Service Escalations

Service escalations are optional follow-up notifications, that generates if a state change is not revoked in time.

Host Dependencies

[Host dependencies] are optional settings to associate multiple hosts.

Service Dependencies

Service dependencies are optional settings to associate multiple services.

External Commands

The interface will help you create your own external commands by generating an example to explain of the supported command you want to use.

Distributed Monitoring

Satellites

Satellites are server which check hosts and deliver the results as a passive check to nagios.

Reporting

Instant Report

Instant reports generate reports in different formats of hosts or hosts and their services in a defined time span with hard or hard and soft sates.

Downtime Report

Downtime reports generate reports in different formats of hosts or hosts and their services in a defined time span in which downtimes occurred.

Current State Report

Current state reports generate in different formats the current service states.

Autoreport

An autoreport automatically reports a report in a user defined way in a defined time to the defined user.

Settings

General settings for an autoreport.
